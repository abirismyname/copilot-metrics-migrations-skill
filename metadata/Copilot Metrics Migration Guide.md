
# GitHub Copilot Metrics Migration Guide

Customer-facing guide for migrating from deprecated **/usage** endpoints to the modern **Copilot Usage Metrics** reporting APIs.

---

# Overview

GitHub is deprecating older **Copilot usage APIs** in favor of the newer **Copilot Usage Metrics** APIs and report endpoints.  
The new APIs provide richer engagement metrics and better breakdowns across editors, languages, models, and repositories.

This guide explains:

- What changed
- How to migrate
- Example analytics use cases
- Architecture patterns for reporting

# Copilot Metrics for Enterprise 

## Key Differences from the Usage API

| Category | Old API – Enterprise Usage | New API – Copilot Metrics |
|---|---|---|
| Date Field | day | date |
| User Metrics | total_active_users | total_active_users + total_engaged_users |
| Code Suggestions | total_suggestions_count | total_code_suggestions |
| Breakdown | editor, language | editor, language, model, repository |
| Chat Metrics | chat_turns | insertions, copy events |
| PR Metrics | Not available | PR summaries created |
| Model Insights | Not available | custom model training date |

# Pulling enterprise team member info (for team-level reporting)

Some reporting workflows require mapping **enterprise teams → members** (for example, to join Copilot usage metrics to team rosters). The GitHub CLI (`gh`) plus `jq` makes it easy to export a CSV of team memberships.

## Prerequisites

- `gh` authenticated with a token that can access the enterprise teams endpoints *
- `jq` installed
- Environment variables set:
  - `ENTERPRISE` (enterprise slug)
  - `OUT` (output CSV path)

* Call `gh auth refresh -h github.com -s admin:enterprise` before running the scriptto give the user the proper permissions.
## Output format

The script writes CSV rows as:

| Column | Description |
|---|---|
| login | GitHub username |
| team_slug | Enterprise team slug |
| team_name | Enterprise team display name |

## Script (gh + jq)

```bash
# 1) list enterprise teams
gh api -H "Accept: application/vnd.github+json" \
  "/enterprises/${ENTERPRISE}/teams" --paginate \
| jq -r '.[] | [.id, .slug, .name] | @tsv' \
| while IFS=$'\t' read -r team_id team_slug team_name; do
    # 2) list members for each enterprise team
    gh api -H "Accept: application/vnd.github+json" \
      "/enterprises/${ENTERPRISE}/teams/${team_id}/memberships" --paginate \
    | jq -r --arg team_slug "$team_slug" --arg team_name "$team_name" \
      '.[] | [.login, $team_slug, $team_name] | @csv'
  done >> "$OUT"

echo "Wrote: $OUT"
```
